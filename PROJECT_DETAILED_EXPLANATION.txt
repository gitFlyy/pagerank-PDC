Scalable PageRank Engine - Detailed Project Explanation
=====================================================

Author in source: Humayun
Course context in source: Parallel and Distributive Computing (CS 3006)


1) What This Project Does
-------------------------
This project builds a distributed PageRank solver in C using MPI, with three communication strategies:

1. Scenario 1: Point-to-point blocking communication (MPI_Send, MPI_Recv)
2. Scenario 2: Collective communication (MPI_Allgatherv)
3. Scenario 3: Asynchronous overlap (MPI_Isend, MPI_Irecv, MPI_Waitall)

The input graph is a directed edge list in a custom .edges format. The project also provides a Python converter that turns web-Google.txt into:

1. METIS adjacency format for partitioning support
2. A remapped directed .edges file for PageRank computation
3. A node ID mapping file


2) End-to-End Data Flow
-----------------------
Step A: Convert dataset
- Script: convert.py
- Reads web-Google.txt
- Writes:
  - web-Google.metis
  - web-Google.metis.mapping
  - web-Google.metis.edges

Step B: Build binaries
- Makefile builds:
  - pagerank_p2p
  - pagerank_collective
  - pagerank_async

Step C: Run with mpirun
- Example:
  mpirun -np 4 ./pagerank_p2p web-Google.metis.edges

Step D: Runtime pipeline inside C program
1. MPI initialized
2. Rank 0 loads whole graph
3. Graph partition array built and broadcast
4. Data distributed to each rank
5. Vertices classified into internal/boundary
6. Selected scenario runs iterative PageRank
7. Rank 0 prints timing + top 10 vertices


3) Core Algorithm: PageRank
---------------------------
For each vertex i:
PR_new(i) = (1 - d) / N + d * sum over all incoming neighbors j of i: PR(j)/out_degree(j)

Where:
- d is damping factor (0.85 in this code)
- N is total number of vertices
- incoming neighbors of i are all j such that edge j -> i exists

Convergence in this project:
- Residual per rank = sum of abs(PR_new - PR_old) over local vertices
- Global residual = MPI_Allreduce(sum across ranks)
- Stop when global residual < 1e-7, or after 200 iterations

4.1 convert.py
--------------
Purpose:
- Preprocess raw edge list into formats needed for partitioning and PageRank runs.

Important concepts used:
- Python dictionaries/sets for remapping and adjacency building
- ID normalization: original IDs -> contiguous IDs [0, N-1]
- Writing both undirected and directed outputs for different tasks

Function: convert(input_file, output_file)
What it does in order:
1. Reads raw edge list; ignores comment lines and malformed lines
2. Collects all nodes and edges
3. Remaps node IDs to dense range [0, N-1]
4. Builds undirected adjacency for METIS output
5. Writes METIS format file:
   - first line: N and undirected edge count
   - one line per vertex: 1-based neighbor IDs
6. Writes mapping file old_id -> new_id
7. Writes directed .edges file for PageRank:
   - first line: N E
   - then each edge u v in remapped IDs

Why directed and undirected outputs both exist:
- METIS graph format expects undirected adjacency
- PageRank needs directed in-neighbors and out-degrees

Main block behavior:
- No args: defaults to web-Google.txt -> web-Google.metis
- Two args: uses provided input and output
- Otherwise prints usage and exits

4.5 src/graph_utils.c
---------------------

Function: Graph* load_graph(const char *edges_file)
Purpose:
- Load directed graph and build CSR for outgoing and incoming edges.

How it works:
1. Read header N and E
2. Read E edge pairs into temporary arrays src/dst
3. Compute out_degree, ignoring self-loops
4. Build outgoing CSR (row_ptr, col_idx)
5. Compute in-degree and build incoming CSR (in_row_ptr, in_col_idx)
6. Free temporaries
7. Return heap-allocated Graph

Important notes:
- Self loops are skipped in degree/csr contribution
- Uses two-pass CSR construction pattern

Function: void free_graph(Graph *g)
Purpose:
- Free all heap arrays in Graph and the Graph object.

Function: idx_t* partition_graph(Graph *g, int num_ranks, MPI_Comm comm)
Purpose:
- Produce ownership array partition[i] = owner rank for vertex i.

Actual behavior in this code:
- Uses linear block partition, not true ParMETIS partitioning
- Rank 0 computes contiguous chunks
- Broadcasts global_part to all ranks

Why linear partition might be used:
- Deterministic and simple
- Works even if advanced partitioning calls are unavailable
- May reduce partition quality vs real graph cut minimization

Function: LocalGraph* distribute_graph(Graph *g, idx_t *partition, ...)
Purpose:
- Build each rank's local graph data.

Major steps:
1. Broadcast N
2. Count vertices per rank
3. Send global vertex IDs to each rank
4. Broadcast full out_degree array to all ranks
5. Build local incoming CSR for each rank from global incoming CSR
6. Rank 0 sends remote ranks their local CSR slices
7. Initialize pr to uniform 1/N

Design choice:
- out_degree is replicated globally on each rank to simplify PR update.

Function: void classify_vertices(LocalGraph *lg, int rank, int num_ranks)
Purpose:
- Split local vertices into internal and boundary.

Definitions:
- Internal vertex: all incoming neighbors are local
- Boundary vertex: at least one incoming neighbor is remote

How it determines locality:
- Binary search of neighbor global ID in sorted local global_ids

Then it builds ghost list:
1. Collect remote neighbor IDs from boundary vertices
2. Sort IDs
3. Deduplicate
4. Store in ghost arrays

Note:
- ghost_owner_rank is declared in struct but not populated/used in this implementation.

Function: void free_local_graph(LocalGraph *lg)
Purpose:
- Free all local graph buffers and arrays.

Function: void print_stats(int rank, int iter, double residual)
Purpose:
- Rank 0 prints iteration number and residual.


4.6 src/pagerank_p2p.c (Scenario 1)
------------------------------------
Function: pagerank_p2p(...)
Communication model:
- Root-mediated manual exchange every iteration.

Per iteration:
1. Rank 0 receives local PR pieces from all ranks
2. Rank 0 reconstructs full global_pr
3. Rank 0 sends full global_pr to all ranks
4. Every rank computes local PR update from global_pr
5. MPI_Allreduce sums residual
6. Swap pr and pr_new
7. Stop on threshold

Pros:
- Easy to reason about
- Explicit data movement

Cons:
- Root bottleneck
- Multiple sends/receives per iteration
- Poor scalability for large rank counts


4.7 src/pagerank_collective.c (Scenario 2)
-------------------------------------------
Function: pagerank_collective(...)
Communication model:
- Uses MPI_Allgatherv so all ranks receive all local PR partitions.

Setup once:
1. MPI_Allgather local_n into counts
2. Build displacements
3. MPI_Allgatherv global_ids into all_gids

Per iteration:
1. MPI_Allgatherv local PR values into gathered_pr
2. Remap gathered order into global_pr indexed by global vertex ID
3. Local compute over incoming neighbors
4. Global residual with MPI_Allreduce
5. Swap arrays and convergence check

Pros:
- Simpler and typically faster than manual root gather+broadcast
- No single root handling all traffic explicitly

Cons:
- Still all-to-all style synchronization
- Full PR vector replicated on every rank


4.8 src/pagerank_async.c (Scenario 3)
--------------------------------------
Function: pagerank_async(...)
Communication model:
- Non-blocking point-to-point with overlap.

Setup:
- Same counts/displs/all_gids setup as collective case
- Allocate send/recv buffers for each rank

Per iteration phases:
1. Post Irecv from every rank
2. Fill local send buffer and post Isend to every rank
3. Compute internal vertices while communication is in flight
4. Waitall for all outstanding requests
5. Reconstruct global_pr from received buffers
6. Compute boundary vertices
7. Allreduce residual, swap, convergence

Goal:
- Hide communication time behind useful computation.

Observed caveat in your runs:
- Converges in 2 iterations, much faster than other scenarios.
- This may indicate algorithmic or synchronization behavior differences that should be validated for numerical equivalence.


5) Concepts and Data Structures in Depth
----------------------------------------

5.1 CSR (Compressed Sparse Row)
- row_ptr has length N+1
- col_idx has length number of stored edges
- Edges of vertex i are in col_idx[row_ptr[i] ... row_ptr[i+1]-1]

Why CSR:
- Compact memory for sparse graphs
- Fast row-wise traversal for neighbor iteration

This project uses two CSR views:
- outgoing CSR (for out-degree and potential traversal)
- incoming CSR (for PageRank updates, since formula sums over in-neighbors)


5.2 Partitioning and Ownership
- partition[i] says which rank owns global vertex i
- local_n is count of owned vertices on a rank
- global_ids maps local index -> global vertex ID

Why ownership matters:
- Only owner stores and updates PR state for that vertex locally
- Other ranks needing remote vertex values depend on communication


5.3 Internal vs Boundary Vertices
- Internal: update can be done from local data only
- Boundary: depends on remote rank values

Why classify:
- Enables overlap in async strategy
- Internal updates can proceed before remote values arrive


5.4 Residual and Convergence
Residual used:
- L1 difference sum over all vertices between current and next PR vectors

Global residual uses MPI_Allreduce with MPI_SUM.

Why this is common:
- Simple global stopping criterion
- Robust for distributed iterations


6) MPI APIs Used and Why
------------------------

MPI_Init / MPI_Finalize
- Start and end MPI runtime.

MPI_Comm_rank / MPI_Comm_size
- Get rank ID and world size.

MPI_Bcast
- One-to-all distribution (N, partition, out_degree).

MPI_Barrier
- Synchronize ranks at phase boundaries (mostly for clean timing/ordering).

MPI_Send / MPI_Recv
- Blocking point-to-point transfer in Scenario 1 and graph distribution.

MPI_Isend / MPI_Irecv
- Non-blocking communication in Scenario 3 to overlap communication and compute.

MPI_Waitall
- Wait until all posted non-blocking operations complete.

MPI_Allgather / MPI_Allgatherv
- All ranks collect each rank's data.
- Allgatherv supports variable data sizes per rank.

MPI_Gather / MPI_Gatherv
- Collect local final arrays to rank 0 for output.

MPI_Allreduce
- Sum residual across ranks for convergence test.

MPI_Wtime
- Wall-clock timer for elapsed runtime.

MPI datatype/constants seen:
- MPI_INT, MPI_DOUBLE
- MPI_SUM
- MPI_COMM_WORLD
- MPI_STATUS_IGNORE, MPI_STATUSES_IGNORE


7) OpenMP in This Project
-------------------------
No OpenMP directives are currently used in the source code.

What this means:
- There are no lines like #pragma omp parallel for.
- Parallelism here is process-level distributed memory (MPI), not thread-level shared memory.

If OpenMP were added, likely targets would be loops over local vertices:
- for (int li = 0; li < lg->local_n; li++) { ... }

Example of what an OpenMP directive would look like (not in current code):
#pragma omp parallel for reduction(+:local_residual)
for (int li = 0; li < lg->local_n; li++) {
    ...
}

12) Practical Observations and Engineering Tradeoffs
----------------------------------------------------

Memory tradeoff:
- Full global PR vector is replicated on every rank in all three scenarios.
- Simpler indexing and implementation, but higher memory usage.

Communication tradeoff:
- Scenario 1: explicit, but root bottleneck
- Scenario 2: concise and balanced, still synchronized each iteration
- Scenario 3: attempts overlap to hide latency

Partitioning tradeoff:
- Current linear partition is easy and deterministic
- Real graph partitioning can reduce edge cuts and communication volume

15) Short Glossary
------------------

- Communicator: process group (MPI_COMM_WORLD)
- CSR: compressed sparse row graph representation
- Internal vertex: all required neighbors local
- Boundary vertex: needs remote neighbor values
- Ghost vertex: remote vertex value needed locally
- Residual: convergence difference metric
